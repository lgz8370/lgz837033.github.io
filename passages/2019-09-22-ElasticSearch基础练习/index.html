# 6.检索查询
  • 参数格式:

GET /user/_doc/_search { "query":{   {检索方式}:{field:value}    } }

  • 检索方式:

term表示精确匹配,value值不会被分词器拆分,按照倒排索引匹配,一般用于filter中 match_phrase表示短语检索,value值不会被分词器拆分,直接去倒排索引中匹配 match表示全文检索,value值会被分词器拆分,然后去倒排索引中匹配 range表示范围检索,有 gt/gte/lt/lte 等

# 6.1 查询名称为'wang da'的员工
# 查询名称为'wang da'的员工
GET /user/_search
{
  "query":{
    "term":{
      "name":"wang"
    }
  }
}

# 10. 逻辑查询

逻辑查询规则: must/should/must_not,相当于 and/or/not

  • 参数格式:

# 10.1 检索部门为1,并且大于32岁的员工
#需求:获取部门为1,并且大于32岁的员工
GET /user/_search
{
  "query": {
    "bool": {
      "must": [
        {
          "term": {
            "deptId":1
          }
        },
        {
          "range": {
            "age": {
              "gt": 32
            }
          }
        }
      ]
    }
  }
}

# 11. 过滤查询

filter表示在查询中不涉及评分,并且有缓存,用于精确匹配,filter中的检索方式多为term

GET /user/_search
{
  "query": {
    "bool": {
      "filter": [
        {
          "term": {
            "deptId": 1
          }
        },
        {
          "range": {
            "age":{
              "gt":30
            }
        }
        }
      ]
    }
  }
}

# 12. 分组查询

分组运算:avg/sum/min/max/value_count/stats(执行以上所有功能)

# 12.1 统计各部门平均年龄
#需求:统计各部门平均年龄
GET /user/_search
{
  "aggs": {
    "groupByDeptId":{
      "terms": {
        "field": "deptId",
        "size": 10
      },
      "aggs": {
        "userAge": {
          "avg": {
            "field": "age"
          }
